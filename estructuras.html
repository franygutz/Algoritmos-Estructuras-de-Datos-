Listas Enlazadas

Introducción a las Listas Enlazadas

Una lista enlazada es un conjunto de elementos (nodos) donde cada elemento contiene la posición o dirección del siguiente elemento de la lista. A diferencia de los arrays, los elementos no necesitan estar en posiciones de memoria contiguas.

Cada nodo típicamente tiene dos campos:

  • Dato: El valor almacenado en el nodo.
  • Enlace (Puntero): La dirección del siguiente nodo. El último nodo apunta a NULO.

Ventajas: Inserción y eliminación eficientes, tamaño dinámico.

Desventajas: Acceso secuencial (no directo por índice), mayor uso de memoria por los punteros.

Ejemplo práctico: Una cadena de favores. Cada persona (nodo) sabe a quién le toca ayudar después (enlace).

Pilas (Stacks)

Introducción a las Pilas

Una pila es un tipo especial de lista lineal donde la inserción (apilar o `push`) y la eliminación (desapilar o `pop`) de elementos se realizan solo por un extremo, llamado cima o tope.

Siguen el principio LIFO (Last-In, First-Out): el último elemento en entrar es el primero en salir.

Ejemplo práctico: Una pila de platos. El último plato colocado encima es el primero que se toma.

inicio

Aplicaciones Comunes

  • Manejo de llamadas a funciones (pila de llamadas).
  • Evaluación de expresiones aritméticas (notación polaca inversa).
  • Algoritmos de retroceso (backtracking).
  • Funcionalidad "Deshacer" (Undo) en editores.
  • Verificación de paréntesis balanceados.

Colas (Queues)

Introducción a las Colas

Una cola es una estructura lineal de datos donde las eliminaciones se realizan por un extremo (el frente o `front`) y las inserciones por el otro extremo (el final o `rear`).

Siguen el principio FIFO (First-In, First-Out): el primer elemento en entrar es el primero en salir.

Ejemplo práctico: Una fila para comprar boletos en el cine. La primera persona en llegar es la primera en ser atendida.

procedimiento sacar (E/S punt: frente, final; S tipo_elemento: elemento)
var punt: auxi
inicio
    si no vacia(frente) entonces
        auxi := frente
        elemento := frente->elemento
        frente := frente->sig
        si frente = nulo entonces // Cola quedó vacía
            final := nulo
        fin si
        liberar(auxi)
    fin si
fin_procedimiento

Aplicaciones Comunes

  • Gestión de tareas en sistemas operativos (planificación de procesos).
  • Colas de impresión.
  • Buffers de datos (por ejemplo, en streaming de video).
  • Algoritmos de recorrido de grafos (como Búsqueda en Anchura - BFS).
  • Sistemas de atención al cliente (gestión de turnos).
